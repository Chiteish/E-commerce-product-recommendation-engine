## 🧬 DSA Concepts & Complexity Analysis

This project bypasses standard libraries (like Python's `heapq` or JS `d3.js`) to implement custom data structures from first principles.

| Structure / Algorithm | E-Commerce Purpose | Time Complexity (Best/Worst) | Space Complexity |
| :--- | :--- | :--- | :--- |
| **Trie (Prefix Tree)** | Search bar query autocompletion. | $O(L)$ to traverse / $O(L + S)$ to extract suggestions | $O(\sum \text{len}(words))$ |
| **Adjacency List Graph** | Bipartite interaction network of users & purchases. | $O(1)$ edge addition / $O(U \cdot P_u)$ co-purchase extraction | $O(V + E)$ |
| **Custom Max-Heap** | Priority Queue sorting candidate similarity scores. | $O(\log N)$ push & pop / $O(N)$ heapify build | $O(N)$ |
| **Jaccard Similarity** | Content-based tags comparison. | $O(T_A + T_B)$ set intersection/union | $O(T_A + T_B)$ |
| **Hash Map Indices** | Amortized constant-time lookup maps. | $O(1)$ average / $O(N)$ worst-case collisions | $O(N)$ |

### Time & Space Complexity Derivation:
*   **Top-K Recommendations via Max-Heap vs Sorting**:
    *   *Sorting-based*: Sorting a catalog of size $N$ takes $O(N \log N)$ time.
    *   *Heap-based*: Streaming $N$ items through a bounded heap of size $K$ takes $O(N \log K)$. Since $K \ll N$ (e.g. recommending 5 items out of 10,000 products), the heap approach is significantly faster and saves CPU cycles in high-throughput production environments.
*   **Bipartite Graph Collaborative Filtering (2-Hop)**:
    *   Instead of computing a giant user-to-user correlation matrix $O(U^2 \cdot I)$, we traverse the adjacency list. Start at the active product node $\rightarrow$ hop to users who purchased it ($O(U_{\text{buyers}})$) $\rightarrow$ hop to their other purchased products ($O(P_{\text{purchases}})$). This local neighborhood traversal yields candidate suggestions in milliseconds.

## 🚀 Installation & Running

This project requires **Python 3.8+** and runs out-of-the-box with zero mandatory dependencies.

### Option 1: Running the Python CLI Engine
1.  **Clone the Repository**:
    ```bash
    git clone https://github.com/yourusername/E-Commerce-Product-Recommendation-Engine.git
    cd E-Commerce-Product-Recommendation-Engine
    ```
2.  **Set Up Virtual Environment**:
    ```bash
    # Windows
    python -m venv venv
    .\venv\Scripts\activate
    
    # macOS/Linux
    python3 -m venv venv
    source venv/bin/activate
    ```
3.  **Install Formatting Dependencies (Optional)**:
    ```bash
    pip install -r requirements.txt
    ```
4.  **Run the CLI**:
    ```bash
    python main.py
    ```

### Option 2: Running the Web visualizer
1.  Navigate into the `web_visualizer/` folder.
2.  Open `index.html` directly in any browser (Double-click or drag-and-drop). No local server, node modules, or compile scripts required!

## 🔄 Recommender System Workflow Pipeline

The engine evaluates recommendations using a 4-step pipeline:
1.  **Data Retrieval**: Matches the active customer profile details in $O(1)$ using Hash Map index maps.
2.  **Candidate Selection**:
    *   *Collaborative filtering*: Extracts items co-purchased by lookalikes via the bipartite graph adjacency traversal.
    *   *Content-Based*: Represents user taste preferences as a tag profile set and compares it with catalog items using Jaccard Similarity.
3.  **Ranking (Max-Heap)**: Normalizes and blends the scores ($40\%$ Content + $40\%$ Collaborative + $20\%$ Item Ratings) and pushes items onto a Max-Heap.
4.  **Extraction**: Pops the top-$N$ highest scoring products from the heap root, filtering out items already purchased or in the active cart.
